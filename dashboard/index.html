Manufacturing Operations Dashboard

An interactive Power BI solution for tracking line efficiency and root cause analysis of production downtime.

Project Overview

This comprehensive dashboard provides production supervisors and engineering teams with real-time insights into manufacturing line performance. It merges productivity metrics with quality control logs to isolate inefficiencies.

Key Insights:

  • Efficiency Tracking: Monitors daily performance against standard batch times, isolating trends by product and operator.
  • Dynamic Pareto Analysis: Automatically calculates the "Vital Few" factors causing 80% of total downtime minutes.
  • Error Isolation: Visualizes operator error frequencies to identify targeted training opportunities.

Phase 1: Data Architecture & Modeling

Before building visuals, the raw production logs were cleaned in Power Query and modeled into a robust Star Schema to ensure high performance and accurate filtering.

  • Data Cleaning: Handled null values by replacing empty text fields with "Uncategorized" and empty numeric fields with 0.
  • Fact Table: Fact Line Productivity & Line downtime act as the core numerical tables containing Batch IDs, Dates, and Duration minutes.
  • Dimension Tables: Downtime factors and Products act as lookup tables.
  • Relationships: 1-to-Many (*) relationships established from Dimension tables filtering the Fact tables, with Single cross-filter direction.

Phase 2: Base Productivity Measures

These core DAX formulas establish the baseline KPIs for the executive summary cards and trend lines.

// Calculates total time recorded per batch
Actual Batch Time = SUM('Fact Line Productivity'[Batch time])

// Calculates expected time based on product standards
Standard Batch Time = SUM('Fact Line Productivity'[Products.Standard batch time])

// Core Efficiency Ratio
Efficiency = DIVIDE([Standard Batch Time], [Actual Batch Time], 0)

// Base measure for Pareto charting
Total Downtime = SUM('Line downtime'[Downtime])

Phase 3: The Pareto Logic

Standard Pareto charts fail in Power BI when data contains tied values. This DAX logic forces the calculation to use the exact Description column on the visual and includes an alphabetical tie-breaker so the cumulative line never flatlines.

1. Cumulative Downtime (Running Total)

Cumulative Downtime = 
VAR CurrentDowntime = [Total Downtime]
VAR CurrentDesc = MAX('Downtime factors'[Description])

RETURN
IF(
    ISBLANK(CurrentDowntime), 
    BLANK(),
    CALCULATE(
        [Total Downtime],
        FILTER(
            ALLSELECTED('Downtime factors'[Description]),
            [Total Downtime] > CurrentDowntime || 
            ([Total Downtime] = CurrentDowntime && 'Downtime factors'[Description] <= CurrentDesc)
        )
    )
)

2. Cumulative Percentage

Cumulative % = 
IF(
    ISBLANK([Total Downtime]),
    BLANK(),
    DIVIDE(
        [Cumulative Downtime],
        CALCULATE([Total Downtime], ALLSELECTED('Downtime factors'[Description]))
    )
)

3. Dynamic Color Switch (Conditional Formatting)

This measure dynamically assigns a hex color code based on where the bar sits relative to the 80% cutoff line.

Pareto Bar Color = 
IF(
    [Cumulative %] <= 0.80,
    "#E66C37", // The Orange color for the Top 80% offenders
    "#5C9CE6"  // The Blue color for the remaining 20%
)

Phase 4: Visual Configuration Guide

The exact settings required to assemble the visuals in the Power BI report canvas.

The Pareto Chart

  1. Visual Type: Line and clustered column chart.
  2. Axes Setup:
    • X-axis: Ensure it is strictly the Description column from the Downtime factors table.
    • Column y-axis: Total Downtime.
    • Line y-axis: Cumulative %.
  3. The Sort: Click the three dots (...) on the visual -> Sort axis -> Choose Total Downtime. Click the dots again and ensure Sort descending is selected.
  4. Conditional Formatting: Go to the Format visual tab -> Columns -> Click the fx button next to color. Change Format style to Field value and select the Pareto Bar Color measure.
  5. 80% Reference Line: Analytics Pane -> Constant Line -> Attach to Secondary Y-Axis -> Value: 0.8 -> Style: Dashed.

Supporting Visuals

  • Executive KPIs: Utilize the 'Card' visual to display Efficiency, Total Batch Time, and Total Downtime.
  • Efficiency by Date: Line chart plotting Date on the X-axis and Efficiency on the Y-axis.
  • Breakdowns: Clustered bar charts utilizing Operator and Product dimensions against the Efficiency measure.
